Research Methodology

The technical and analytical approach used to build, train, and evaluate the Predictive Intrusion Detection System.

1 Data Collection & Lab Setup

A predictive model requires high-fidelity, sequential network data. We established a custom lab environment to simulate real-world cyber attacks (DDoS, Brute Force, Port Scans) alongside benign traffic.

Figure 1: Lab topology illustrating raw PCAP capture from mixed traffic sources.

  • Traffic Generation: Utilized Nmap and Kali Linux toolsets to generate realistic attack vectors.
  • Packet Sniffing: Captured raw network traffic using Wireshark and Tshark.
  • Flow Extraction: Converted PCAP files into statistical bidirectional flows (CSV) using CICFlowMeter, generating over 80 specific network features.

2 Feature Engineering & Time-Series Formatting

To enable predictive capabilities, raw data was transformed into a time-aware format. We utilized Sliding Windows to allow the model to analyze sequences of network behavior rather than isolated events.

  • Handling Nulls: Removed Corrupted Records and imputed missing values using median strategies.
  • Min-Max Scaling: Normalized features like Flow Duration and Packet Length to ensure model stability during gradient descent.
  • Target Shifting: Re-labeled the dataset to predict if an attack will occur in the next 60 seconds based on the current behavioral window.

3 Modeling & Training Architecture

We compared traditional Machine Learning baselines against Advanced Deep Learning sequence models. The Long Short-Term Memory (LSTM) network was selected as the primary engine due to its ability to retain temporal context over long network sessions.

4 Explainable AI (XAI) & Evaluation

A "black-box" model is not suitable for critical SOC (Security Operations Center) environments. We integrated Explainable AI to provide analysts with transparent reasoning behind every prediction.

  • SHAP Values: Integrated SHapley Additive exPlanations to visualize which specific network features (e.g., a sudden spike in SYN rates) triggered the predictive alert.
  • Prediction Horizon Metric: Evaluated the model not just on accuracy, but on how early it can successfully predict an attack before the payload is delivered (Target: >30 seconds warning).
